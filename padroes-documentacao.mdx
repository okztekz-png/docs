## Objetivo

Esta página define o padrão mínimo para documentar ambientes técnicos mantidos pela **Tekz Tecnologias**.

O objetivo é garantir que qualquer técnico consiga entender rapidamente:

\* quem é o cliente; \* como a rede está organizada; \* quais equipamentos existem; \* quais sistemas são críticos; \* onde ficam os acessos; \* quais procedimentos devem ser seguidos; \* o que fazer em caso de incidente.

## Regras de segurança

<Warning>
  Nunca registre senhas, tokens, chaves privadas, seeds, códigos de recuperação ou credenciais diretamente na documentação.
</Warning>

Use a documentação apenas para indicar onde a credencial está armazenada.

Exemplo recomendado:

```txt
Acesso ao firewall:
Credencial armazenada no Bitwarden em:
Cliente / Jodano / Firewall Principal
```

Exemplo que deve ser evitado:

```txt
Usuário: admin
Senha: senha123
```

## Padrão de escrita

Sempre que possível, escreva de forma objetiva:

```txt
Servidor principal do cliente.
IP: 192.168.1.253
Função: hospeda o sistema SIAF via Winflector.
Localização: VM dentro do Proxmox.
```

Evite textos vagos como:

```txt
Servidor que fica lá no cliente e roda o sistema.
```

## Campos recomendados para equipamentos

Para roteadores, firewalls, switches, servidores e access points, use o seguinte padrão:

```txt
Nome:
Tipo:
Fabricante/modelo:
IP:
Localização física:
Função:
Acesso:
Observações:
```

```txt
Sistema operacional:

Função:
Serviços instalados:
Backup:

Observações:
```
