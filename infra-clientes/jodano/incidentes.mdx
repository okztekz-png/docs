## Histórico de incidentes

Até o momento, não há incidentes registrados nesta documentação.

## Modelo para registro de incidente

Use o padrão abaixo sempre que houver um problema relevante no ambiente.

```text
Data:
Cliente: Jodano
Problema:
Impacto:
Causa identificada:
Solução aplicada:
Técnico responsável:
Observações futuras:
```

## Exemplos de incidentes que devem ser registrados

- Queda de internet.
- Falha no roteador.
- Falha no switch principal.
- Proxmox indisponível.
- Servidor `192.168.1.253` desligado ou travado.
- Falha no acesso externo ao SIAF.
- Winflector parado.
- Alteração ou perda de regras de encaminhamento de porta.
- Troca de ONU ou roteador.
- Problemas de backup ou restauração.

## Observações

- Todo incidente que afetar operação, sistema SIAF, internet ou acesso externo deve ser registrado.
- Sempre que uma solução for aplicada, documentar o que foi feito para facilitar atendimentos futuros.
