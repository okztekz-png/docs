## Modelo para registro de incidente

Use o padrão abaixo sempre que houver um problema relevante no ambiente.

```text
Data:
Cliente: Colônia Agrícola
Problema:
Impacto:
Causa identificada:
Solução aplicada:
Técnico responsável:
Observações futuras:
```

## Exemplos de incidentes que devem ser registrados

- Queda da WAN 1 / Infortel.
- Queda da WAN 2 / Starlink.
- Failover não funcionando.
- OPNsense indisponível.
- DHCP parado em alguma VLAN.
- Problema de roteamento entre VLANs.
- Switch TP-Link inacessível.
- Porta trunk do AP UniFi com problema.
- AP UniFi offline.
- SSID entregando IP de VLAN incorreta.
- VMware Host indisponível.
- UCS indisponível.
- Nextcloud fora do ar.
- UniFi Controller parado.
- VM Windows inacessível.
- Falha em backup ou restauração.

## Observações

- Todo incidente que afetar internet, VLANs, servidores, Wi-Fi, UCS, Nextcloud ou firewall deve ser registrado.
- Sempre que uma solução for aplicada, documentar o que foi feito para facilitar atendimentos futuros.
- Alterações relevantes também podem ser registradas nesta página enquanto não houver uma página exclusiva de histórico.
